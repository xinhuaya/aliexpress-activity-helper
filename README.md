# 速卖通活动退出助手

这是给速卖通铺货运营使用的浏览器脚本。核心目标是：在商品优化窗口期，直接在任意卖家后台页面一次输入 1-10 个商品 ID，查出这些商品已经报名且未结束的活动，并自动逐项普通退出。

## 当前能力

- 在速卖通卖家后台右下角显示“AE 活动助手”面板。
- 无需手动进入商品管理页或营销活动页；一次可输入 1-10 个商品 ID，每行一个，也支持空格、逗号分隔。
- 批量扫描结果按商品分组显示；重复 ID 会自动去重，没有活动或扫描失败的商品会标记并跳过。
- 直接读取商品管理接口返回的该商品 `SALE` 数据，与鼠标悬浮看到的内容相同。
- `SALE` 同时显示“平台入围”和“店铺外围”时，两类报名分别保留、分别退出，不再把平台活动合并掉。
- 同名平台活动出现多个相近编号时，只核对这些候选中该商品的实际报名记录，再决定退出编号。
- 只为 `SALE` 已显示的活动匹配活动编号，不再遍历全店几十个活动入口。
- 默认跳过已结束活动。
- 退出原因自动使用“库存不足”。
- 按页面自己的按钮流程提交普通“退出活动”，不会设置为“不参加活动商品”。
- 兼容普通活动页和统一报名入口；平台入围活动会自动依次经过外围商品页、店铺资质审核和入围商品页，再搜索商品。
- 进入活动页时如果出现“营销智投计划”介绍弹窗，会自动点击“我知道了”关闭。
- “查报名活动”只读取并展示计划；真正退出前还有一次浏览器确认框。
- 退出队列支持手动“暂停/继续”；按钮缺失、处罚提示或页面结构异常时会自动暂停并停留在当前活动页。
- 检测到 Baxia 安全验证时会暂停并保留当前任务，不会尝试绕过或自动处理验证。

## 文件

- `aliexpress-activity-helper.user.js`：给 Tampermonkey 安装的浏览器脚本。
- `使用说明-运营同事版.md`：给同事看的最短使用说明。
- `scripts/build-release.mjs`：生成 GitHub Pages 更新文件。
- `.github/workflows/deploy-pages.yml`：测试通过后自动发布更新。

## 安装

1. Chrome 安装 Tampermonkey。
2. 右键 Tampermonkey 图标，进入“管理扩展程序”，打开“允许用户脚本”。如果没有该开关，则在 `chrome://extensions` 打开右上角“开发者模式”。
3. 打开 [活动助手安装页](https://xinhuaya.github.io/aliexpress-activity-helper/)。
4. 点击“安装或更新稳定版”，在 Tampermonkey 页面确认安装。
5. 回到速卖通卖家后台，按 `Ctrl + R` 刷新页面。安装后不刷新，脚本不会在已经打开的页面中自动出现。

当前脚本版本为 `0.9.2`，支持最多 10 个商品组成的批量队列，并匹配所有 `aliexpress.com` 子域名。脚本会自动记住当前账号网址里的 `channelId`，不会沿用其他账号的渠道编号，并且只在顶层页面运行，避免同域 iframe 重复显示助手。统一报名入口会等待资质页按钮稳定且可操作后再点击，并耐心等待页面跳转。退出弹窗中的原因选项会等待异步加载，并兼容不同单选框组件结构。

从 `0.8.5` 开始，脚本使用固定 GitHub Pages 地址检查更新。旧版用户需要通过安装页覆盖安装一次；完成这次迁移后，后续版本由 Tampermonkey 自动检查，不需要再复制粘贴脚本。

## 使用

1. 打开任意速卖通卖家后台页面，不需要进入“营销 > 平台活动”。
2. 在右下角面板输入 1-10 个商品 ID，建议每行一个；也可以用空格、英文或中文逗号分隔。
3. 点击“查报名活动”；这一步只读取数据，不会提交退出。
4. 脚本会按顺序读取每个商品的 `SALE` 标签数据，并按商品分组列出活动。10 个商品所需时间取决于各自活动数量。
5. 核对列表后点击“普通退出”。
6. 浏览器只会弹出一次总确认框；确认后脚本按商品和活动顺序自动处理。
7. 脚本会进入“已报名”，搜索对应商品，打开“申请退出活动”，选择“库存不足”，点击普通“退出活动”。
8. 运行中可随时点击“暂停”。脚本确认暂停后会停留在当前活动页面，按钮变为“继续处理”。
9. 找不到退出按钮、发现处罚提示、页面结构异常或遇到安全验证时，脚本会自动暂停，方便人工查看平台提示。异常活动会记入“需复查”，点击“继续处理”后再进入下一条。
10. 全部处理结束后只弹出一次汇总提醒，列出成功数和需要复查的商品。

脚本只处理 `SALE` 标签数据里明确列出的平台活动和店铺活动；这些内容就是鼠标悬浮 `SALE` 时看到的列表。如果商品没有 `SALE` 标志，脚本不会生成退出计划。

活动名称和时间必须能唯一匹配活动编号。某个商品出现无法匹配或一对多时，脚本不会猜测，会标记并跳过该商品，继续扫描其余商品。如果速卖通弹出安全验证，脚本会暂停并保留当前任务，且不会绕过平台验证。

## 安全边界

- 脚本不会绕过验证码、风控或平台限制。
- 脚本不会读取、保存、分享账号密码或 Cookie。
- 页面提示不能退出时，以平台返回结果为准。
- 重新加入活动涉及价格、折扣、SKU 库存等字段，建议在退出功能稳定后，再单独实测并固化“一键重加”。

## 发布维护

1. 修改 `aliexpress-activity-helper.user.js`。
2. 同时提高头部 `@version`、代码中的 `SCRIPT_VERSION` 和 `package.json` 版本，例如 `0.9.1` 改为 `0.9.2`。
3. 运行 `npm run verify`。
4. 提交并推送到 `main` 分支。
5. GitHub Actions 测试通过后自动发布到原来的稳定版网址。

不要更改 `@updateURL` 和 `@downloadURL`。Tampermonkey 通过固定网址和更高的 `@version` 判断是否存在更新。
